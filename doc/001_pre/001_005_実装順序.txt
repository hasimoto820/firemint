================================================================================
方針
================================================================================

  - 箱の作成と中身の実装は別。
    Phase 0 の最初で feature 9個 + shared 全部のフォルダを作る。
    中身は Phase 1 以降、順番に実装する。
  - shared は一括で作らない。必要になった Phase で最小限ずつ増やす。
  - 各 feature の中では main → preload 追記 → renderer の順（main から入る）。


================================================================================
3層の全体像
================================================================================

  Phase 0    土台（Electron・フォルダ・IPC の型）
  Phase 1〜5 MVP feature 順
  Phase 6〜  v1 feature 順


================================================================================
Phase 0：土台（feature より先）
================================================================================

  Step   内容                              Done の基準
  ────   ───────────────────────────────   ─────────────────────────────────
  0-1    electron-vite + React + TS 雛形   npm run dev でウィンドウが開く
  0-2    001_003 どおりにフォルダ組み替え  全 feature / shared の箱あり

--------------------------------------------------------------------------------
Phase 1 — connection
--------------------------------------------------------------------------------

  作るもの:
    shared/firestore          … firebase-admin 接続基盤
    features/connection       … サービスアカウント JSON 読み込み・接続
    shared/safety（最小）     … 本番警告・環境ラベル

  Done:
    □ サービスアカウント JSON で Firestore に接続できる


--------------------------------------------------------------------------------
Phase 2 — explorer
--------------------------------------------------------------------------------

  作るもの:
    shared/shell（最小）      … レイアウト・サイドバー枠
    shared/ui（最小）         … テーブル・ボタン等
    features/explorer         … コレクション一覧、Table、JSON、ドキュメント CRUD

  Done:
    □ コレクション一覧 → テーブル表示 → JSON 編集ができる
    □ ドキュメント CRUD（作成・更新・削除）
    □ サブコレクションの閲覧（最低限）


--------------------------------------------------------------------------------
Phase 3 — query
--------------------------------------------------------------------------------

  作るもの:
    features/query            … Simple クエリ、Collection Group
    （結果表示は explorer のテーブルを再利用）

  Done:
    □ where 条件付きクエリが実行できる（where 1〜3、orderBy、limit）
    □ Collection Group クエリが動く


--------------------------------------------------------------------------------
Phase 4 — bulk_operations
--------------------------------------------------------------------------------

  作るもの:
    shared/safety（拡張）     … 破壊的操作確認、件数・概算コスト表示
    features/bulk_operations  … 一括削除、一括更新、diff プレビュー、バッチ 500
    （explorer / query との協調）

  Done:
    □ テーブルで複数選択 → 一括削除できる
    □ クエリ結果のフィールドを一括更新できる
    □ 一括更新前に diff プレビューが見える
    □ バッチ 500 件制限を UI で表示・自動分割


--------------------------------------------------------------------------------
Phase 5 — data_transfer
--------------------------------------------------------------------------------

  作るもの:
    features/data_transfer    … JSON / CSV エクスポートのみ（import は v1）

  Done:
    □ コレクション → JSON エクスポート
    □ クエリ結果 → JSON / CSV エクスポート
    □ 001_002 の MVP Done 基準をすべて満たす → β版として使える状態

================================================================================
各 feature の中での実装順（毎回同じ型）
================================================================================

  1. shared/    … 型・定数
  2. main/      … 処理本体 + IPC handler
  3. preload    … API 露出（薄く追記）
  4. renderer/  … UI

  main から入る。UI だけ先に作ると IPC が後追いになりがち。


================================================================================
Phase 6〜：v1（MVP 後）
================================================================================

  v1 の最初は shared/ui 整理（001_006 反映）。機能追加の前に横断 UI を共通化する。
  以降は依存の少ない・体験インパクトの大きい順。順番は v1 着手前に再確認可。


  Phase   内容
  ──────  ─────────────────────────────────────────────────────

  7       workspace              … 複数プロジェクト、ラベル、read-only
  8       explorer 拡張          … Tree、フィルタ、複製、Geopoint 等
  9       shared/shell 拡張      … タブ、Split View 枠、Command Palette
  10      query 拡張             … Saved Queries、starts-with、オートコンプリート
  11      data_transfer 拡張     … import、全 DB バックアップ、プロジェクト間転送
  12      connection 拡張        … Google サインイン、Emulator 対応
  13      script_runner          … JS エディタ、Background Tasks
  14      auth                   … Firebase Auth ユーザー管理
  15      onboarding             … 初回オンボーディングガイド（日本語）
  16      shared/i18n, settings  … 日本語 UI 整備、ダークテーマ、ショートカット

Phase 6 — shared/ui 整理
--------------------------------------------------------------------------------

  作るもの:
    shared/ui/                … 横断 UI 部品の移動・集約
    （001_006 参照）
      EnvironmentBadge        … connection/renderer から移動
      DocumentTable           … explorer/renderer から移動
      DocumentJsonPanel       … explorer/renderer から移動
    features/*/renderer       … import 先を shared/ui に更新（機能変更なし）

  方針:
    - 部品（見た目）→ shared/ui/
    - Firestore の取得・更新・ページ構成 → feature/ のまま
    - DocumentSummary 等の型は当面 feature/shared または shared 側で整理（移動時に決める）

  Done:
    □ EnvironmentBadge / DocumentTable / DocumentJsonPanel が shared/ui/ にある
    □ explorer / query の import が shared/ui 経由
    □ 既存 MVP 機能（表示・選択・一括操作）に退行がない
