================================================================================
方針
================================================================================

  - 箱の作成と中身の実装は別。
    Phase 0 の最初で feature 9個 + shared 全部のフォルダを作る。
    中身は Phase 1 以降、順番に実装する。
  - shared は一括で作らない。必要になった Phase で最小限ずつ増やす。
  - 各 feature の中では main → preload 追記 → renderer の順（main から入る）。

================================================================================

  Phase 0    土台（Electron・フォルダ・IPC の型）
  Phase 1〜5 MVP feature 順
  Phase 6〜  v1 feature 順
