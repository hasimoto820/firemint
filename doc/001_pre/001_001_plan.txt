■ プロダクト概要
  Firestore 外部管理デスクトップツール。
  公式コンソールの不足（一括検索・一括更新など）を補う。
  Firefoo 系の後発。価格はやや安く、使いやすさで差別化。


================================================================================
技術選定
================================================================================

【採用】Electron + TypeScript + React（フロント）

【不採用】Tauri
  → 2025-07 時点で撤退。理由は下記「Tauri 撤退メモ」参照。


--------------------------------------------------------------------------------
スタック詳細
--------------------------------------------------------------------------------

  レイヤー          技術
  ─────────────────────────────────────────
  デスクトップ      Electron
  フロント UI       React + TypeScript
  Firestore 接続    firebase-admin（main process）
  UI コンポーネント  未定（Ant Design / shadcn 等は実装時に決定）
  ビルド            electron-vite または electron-forge
  対象 OS           MVP: Windows → v1: Mac / Linux

  アーキテクチャ:

    Renderer (React)
        │  IPC（contextBridge）
        ▼
    Main Process (Node.js)
        └── firebase-admin
              ├── Firestore CRUD / クエリ / バッチ
              ├── Firebase Auth（v1）
              └── JS クエリエディタ実行（v1）


--------------------------------------------------------------------------------
Electron を選んだ理由
--------------------------------------------------------------------------------

  1. firebase-admin が Node 専用 → main process でそのまま使える
  2. Firefoo と同じ路線 → 参考実装・パターンを真似しやすい
  3. JS クエリエディタ（v1）が自然に実装できる（admin SDK 直実行）
  4. lodash, csv 等の npm エコシステムがそのまま使える
  5. MVP 到達速度が最速（Tauri + Node sidecar より単純）


--------------------------------------------------------------------------------
Tauri 撤退メモ（将来参照用）
--------------------------------------------------------------------------------

  検討したが不採用。弱点:
  - Admin SDK を WebView から直接使えない（Node sidecar 必須）
  - sidecar にすると IPC 層が増え、Electron と比べて複雑さだけ増す
  - Firestore 向け Tauri プラグイン・実装例がほぼ無い
  - JS エディタ＋npm 連携が Electron より大幅に面倒

  軽量化は v2 以降の検討事項。v1 は開発速度優先。
