================================================================================
17-a  Emulator 接続
================================================================================

  やること:
    Firebase Local Emulator Suite（Firestore Emulator）に FireMint から接続する。
    ホスト（例: 127.0.0.1:8080）を指定し、workspace に載せる。

  作る場所（想定）:
    features/connection/     … 接続 UI（JSON / Google / Emulator の選択）
    shared/firestore/        … Emulator ホスト設定・接続プールへの載せ方
    features/workspace/      … 名簿に Emulator 由来のエントリをどう表すか

  FireFoo 参考:
    - Add Local Emulator（HOST:PORT）
    - 起動中 Emulator の検出
    - 実プロジェクト接続が先、という制約の有無は要確認

  未策定:
    □ 接続 UI の置き場（接続画面 / workspace 追加フロー）
    □ projectId をどう扱うか（Emulator 起動時の projectId と workspace id）
    □ 自動検出（Discover）を v1 でやるか、手動 HOST:PORT のみか
    □ read-only デフォルトをどうするか


================================================================================
17-b  プロジェクト間転送（dev ↔ prod 等）
================================================================================

  やること:
    workspace でロード済みの 2 つの projectId 間で、コレクション単位に
    データをコピーする（ファイルを人手で経由しなくてもよい）。

  ざっくりフロー:
    1. ソース projectId + collectionPath
    2. 宛先 projectId + collectionPath（同名 or リネーム）
    3. オプション（サブコレクション含むか、衝突方針）
    4. プレビュー（ソース件数・宛先の既存件数）
    5. 実行 → レポート

  実装イメージ:
    - 内部的には「ソース読取 →（任意で一時）→ 宛先書込」
    - UI 上は transfer 専用。ユーザーに raw JSON を触らせないのが主目的
    - ソース＝宛先の同一 projectId は拒否

  001_002 の Saved Data Transfers:
    転送設定（ソース/宛先/path/オプション）の名前付き保存。
    Phase 17 初期は必須にしない。必要なら後半で saved_queries と同様に
    config/ 配下へ（workspaces.json には入れない）。

  未策定:
    □ 単位: コレクションのみ / ドキュメント複数選択 / クエリ結果転送
    □ サブコレクションを含めるデフォルト
    □ 衝突方針（import と共通化するのがよい）
    □ 宛先が非 read-only であることの確認（detectEnvironment は使わない方針）
    □ 双方向同期（diff）は対象外でよいか → v1 は単方向コピー推奨
    □ 失敗時の部分ロールバックはしない（レポートのみ）でよいか

17-c  Emulator 同期
================================================================================

  やること:
    Emulator と実プロジェクト（または別 Emulator）のあいだでデータを動かす。

  依存:

    - 本項目は、接続できたあとの「データを移す」側

  位置づけ（推奨）:
    - 技術的には「17-b プロジェクト間転送」のソース/宛先の一方が Emulator のケース
    - 専用 UI を厚くするか、転送 UI に Emulator project が並ぶだけで足りるかは未定
    - 17-a 前に単独実装すると空振りする → 17-a → 17-b のあと、または 17-b に吸収
