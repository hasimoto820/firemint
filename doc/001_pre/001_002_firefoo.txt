
FireFoo 機能調査 & FireMint 優先度マップ

■ 参考: Firefoo 料金（2025時点）

================================================================================
A. Firefoo 機能一覧（競合リファレンス）
================================================================================

【接続】
  - Google アカウントでサインイン
  - サービスアカウント JSON で接続
  - Firebase Local Emulator Suite 対応
  - データ・認証情報はローカル完結（Secure by design）

【プロジェクト管理】
  - 複数プロジェクト同時ロード
  - プロジェクトに色分けラベル
  - プロジェクトを read-only モードに設定（誤操作防止）

【アプリ基盤】
  - デスクトップアプリ（Mac / Windows / Linux）

【データ閲覧・編集（Firestore）】
  - サイドバーでコレクション一覧
  - タブで複数コレクションを開く
  - Split View（左右にコレクション並列表示）
  - Table / Tree / JSON の3表示モード
  - サブコレクション CRUD（作成・閲覧・編集・複製・移動・削除）
  - ドキュメントの作成・編集・複製・削除
  - コレクションのリネーム（全ドキュメント＋サブコレクション再帰コピー）
  - コレクション・ドキュメントの複製
  - テーブル: カラムの表示/非表示・並べ替え
  - テーブル: ヘッダークリックでソート
  - テーブル: ヘッダー/セル右クリックでフィルタ
  - テーブル: 複数ドキュメント選択 → 一括削除
  - テーブル: フィールド名のリネーム・削除（コレクション全体）
  - 出力オプション（Timestamp / Geopoint / 数値の表示形式カスタム）
  - createTime / updateTime / readTime カラム表示
  - Geopoint を地図上にプロット
  - Firestore 内画像のプレビュー
  - ドキュメント比較（Split View ベース）

【クエリ】
  - Simple エディタ（パス・limit・where・orderBy をフォーム入力＋オートコンプリート）
  - JavaScript エディタ（Node.js Admin SDK、VS Code ベースのシンタックスハイライト）
  - JS エディタ: 組み込み npm パッケージ（fetch, fs, moment, csv, lodash, uuid, jwt 等）
  - Collection Group クエリ
  - starts-with 演算子
  - 手動 Run 実行（リアルタイムリスナーなし）
  - クエリの保存・復元（Saved Queries）
  - Command Palette（Cmd/Ctrl+P）でコレクション・クエリ・設定へジャンプ

【一括操作・スクリプト】
  - JS スクリプトで一括 read / write / delete
  - バッチ書き込み（500件単位）のサンプル・パターン
  - Background Tasks（長時間エクスポート等をバックグラウンド実行）
  - Tasks ポップアップで進捗確認

【インポート / エクスポート / 転送】
  - CSV / JSON インポート（フィールド名・ドキュメントID・型変換の細かい制御）
  - CSV / JSON エクスポート（コレクション・ドキュメント・フィールド単位）
  - 全 DB バックアップ（圧縮アーカイブ）
  - プロジェクト間データ転送（dev ↔ prod ↔ emulator）
  - Saved Data Transfers（転送設定の保存）

【Firebase Authentication】
  - ユーザー一覧・検索・ソート
  - メール・パスワード・電話番号の編集
  - custom claims（カスタムトークン）の編集
  - email verified 等のフィールド編集
  - サインアップ/サインイン時刻の分単位表示
  - ユーザーの一括削除・無効化
  - CSV / JSON でユーザーの import / export
  - プロジェクト間ユーザー転送
  - Auth 用 JS スクリプト

【UX・その他】
  - ダークテーマ
  - キーボードショートカット
  - タブのドラッグ&ドロップ並べ替え
  - 14日間無料トライアル

【Firefoo にない / 弱い（競合 Fuego 等との比較より）】
  - マルチデータベース対応（Firestore の (default) 以外）
  - OR / AND 複合クエリ UI
  - between 演算子
  - リアルタイム更新（クエリ結果の live reload）
  - bytes 型のネイティブ表示
  - 保存時のコンフリクト検出
  - Point-in-Time Recovery (PITR)
  - クエリの explain / analyze
  - Auth マルチテナンシー
  - Firebase Storage 管理
  - カスタムダッシュボード


================================================================================
B. FireMint 優先度マップ
================================================================================

凡例:
  MVP   = 最初のリリース。これだけで「公式よりマシ」と言える最小セット
  v1    = MVP 後すぐ。Firefoo っぽさの核。差別化の土台
  後回し = 需要はあるが工数大 or ニッチ。v1 安定後


--------------------------------------------------------------------------------
【MVP】— 「接続して、見て、検索して、まとめて直せる」
--------------------------------------------------------------------------------

接続
  [MVP] サービスアカウント JSON で接続
  [MVP] (default) データベースのみ

プロジェクト管理
  [MVP] 単一プロジェクト接続（複数は v1）

データ閲覧
  [MVP] サイドバー: ルートコレクション一覧
  [MVP] コレクションをタブで開く
  [MVP] Table ビュー（ドキュメント一覧）
  [MVP] JSON ビュー（ドキュメント詳細の閲覧・編集）
  [MVP] ドキュメント CRUD（作成・更新・削除）
  [MVP] サブコレクションの閲覧（Tree またはインライン展開で最低限）

クエリ・検索
  [MVP] Simple クエリ UI（where 1〜3条件、orderBy、limit）
  [MVP] 手動 Run 実行
  [MVP] Collection Group クエリ（よく使うので早めに）

一括操作 ★本命★
  [MVP] テーブルで複数選択 → 一括削除
  [MVP] クエリ結果に対する一括フィールド更新（プレビュー付き）
  [MVP] 一括更新前の diff プレビュー（FireMint 差別化の核）
  [MVP] バッチ書き込み（500件制限を UI で表示・自動分割）

エクスポート（最低限）
  [MVP] コレクション → JSON エクスポート
  [MVP] クエリ結果 → JSON / CSV エクスポート

安全・UX
  [MVP] 本番モード警告（接続時に環境ラベル表示）
  [MVP] 破壊的操作の確認ダイアログ
  [MVP] 読み取り/書き込み件数・概算コストの表示（一括操作時）

技術基盤
  [MVP] デスクトップアプリ（Electron — firebase-admin）
  [MVP] Windows 対応（Mac は v1 で追随可）

================================================================================
C. FireMint 差別化メモ（Firefoo の弱点を突く）
================================================================================

1. 一括更新の「安全さ」
   → 更新前 diff プレビュー、件数・コスト表示、本番モード二重確認
   → Firefoo はスクリプト任せになりがち。GUI で安心して触れるのが勝ち

2. 日本語
   → UI・エラー・ドキュメント全部日本語。Firefoo は英語のみ

3. 価格
   → 個人 $5〜7/月、チームは Firefoo の 60〜70% を目安

4. オンボーディング
   → 初回起動で「サービスアカウントの作り方」ガイド内蔵（日本語）
   ※ 軽量化は v2 検討（現状 Electron 採用。Firefoo と同系統）

5. コンフリクト検出（v1）
   → 同時編集でデータを壊さない。Fuego にはあって Firefoo にはない


================================================================================
D. MVP 完了の定義（Done の基準）
================================================================================

  □ サービスアカウントで Firestore に接続できる
  □ コレクション一覧 → テーブル表示 → JSON 編集ができる
  □ where 条件付きクエリが実行できる
  □ クエリ結果を選択して一括削除できる
  □ クエリ結果のフィールドを一括更新でき、更新前にプレビューが見える
  □ JSON / CSV エクスポートができる
  □ 破壊的操作に確認が入る

  → 上記が動けば「β版として使える」＝ 早期ユーザーを募れる状態
